# Design Maker 页面缩略导航方案

## 交互原型（低保真）

```text
┌───────────────────────────────────────────────────────────────────────┐
│ 画布区域                                                              │
│                                                                       │
├───────────────────────────────────────────────────────────────────────┤
│ 缩略导航栏                                                            │
│ [◀] [页1缩略] [页2缩略] [页3缩略] ... [页N缩略] [▶]                 │
│       ↑ 当前高亮（2px 蓝色边框 + 阴影）                              │
└───────────────────────────────────────────────────────────────────────┘
```

移动端（<=640px）：

```text
┌─────────────┐
│ 缩略导航栏   │
│ [◀]      [▶] │
│ [页1缩略]     │
│ [页2缩略]     │
│ [页3缩略]     │
└─────────────┘
```

## 技术实现说明

- 组件：`design-thumbnail-nav.ts`
  - 虚拟窗口计算：`computeThumbnailVirtualWindow(...)`
  - 键盘索引计算：`getNextThumbnailIndex(...)`
  - 同步滚动定位：`ensureActiveItemVisible(...)`
- 视图接入：`design-maker-view.ts`
  - 统一切页入口：`_setActivePage(...)`
  - 键盘左右切页：`_switchActivePage(...)`
  - 左侧列表与底部缩略栏共用 `activePageType`，保证双向同步高亮
- 样式：`styles.css`
  - 高亮统一：2px 边框 + 阴影
  - 箭头滚动、拖拽滚动、响应式与加载/失败占位样式

## 性能优化

- 当页面数 `>50` 时启用虚拟滚动，只渲染可视窗口 + overscan，显著降低 DOM 数量。
- 切页同步为本地状态更新，常规路径无重计算瓶颈。

## A/B 测试方案

- A 组：仅底部缩略栏（可点击 + 高亮）
- B 组：底部缩略栏 + 箭头滚动 + 拖拽滚动 + 键盘切页
- 指标：
  - 页面切换完成耗时（P50/P95）
  - 用户在 10 次切页任务中的总操作步数
  - 误触率（非目标页点击后撤销）
  - 导航栏可见区域外目标页到达时间
- 结论判定：
  - B 组在 P95 切页耗时下降 >=15%，且误触率不升高时，默认启用 B 方案
