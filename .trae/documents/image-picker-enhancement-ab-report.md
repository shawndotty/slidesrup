# Design Maker 本地图片选择器增强：A/B 方案与实施报告

## 背景
- 目标：为 Design Maker 的本地图片选择器补齐「智能定位 / 置顶 / 拖拽 / 键盘 ESC+Enter / 动画 / 触摸友好 / 响应式」能力。
- 范围：仅改造 `openLocalImagePicker` 交互与样式，不改变图片插入数据结构。

## 方案对比

### A 方案：增强现有轻量弹层（本次落地）
- 设计：继续使用当前自建浮层 DOM，补充智能定位算法、拖拽、键盘行为、置顶状态与动画。
- 优点：
  - 与现有 Obsidian 插件风格一致；
  - 依赖最小，无需新增第三方库；
  - 变更范围小，回归风险较低。
- 缺点：
  - 高阶浮层能力（碰撞检测策略、复杂焦点陷阱）需手动维护。

### B 方案：引入统一浮层基础层（例如 Popper/自研弹层管理器）
- 设计：抽象出通用 Floating Layer，图片选择器仅实现业务内容。
- 优点：
  - 可复用于更多弹窗/菜单；
  - 对复杂定位场景更稳健。
- 缺点：
  - 抽象成本较高；
  - 当前仓库未为此准备统一弹层体系，短期交付成本偏高。

## 选择结论
- 选择 A 方案：满足当前需求且实现速度快，和现有架构耦合最低。
- 同时将定位与索引切换逻辑提取为纯函数，便于后续平滑升级到 B 方案。

## 已实现能力（A 方案）
- 智能定位：根据触发点上下空间自动选择 `top/bottom`，并做视口边界钳制。
- 置顶：支持 pin/unpin；置顶时提高层级并阻止外部点击关闭。
- 拖拽：支持通过头部拖拽，使用 Pointer Events，鼠标与触屏统一。
- 键盘：
  - `Esc` 关闭；
  - `Enter` 选择当前高亮项并插入；
  - `ArrowUp/ArrowDown` 循环切换高亮项。
- 动画：进入/关闭添加轻量过渡（透明度+缩放）。
- 触摸友好：更大点击热区、`touch-action` 与拖拽细节优化。
- 响应式：在窄屏下调整宽度/高度与字号，避免溢出。
